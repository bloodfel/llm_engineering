## 概念理解题

### Q1. Transformer 为什么取代了 LSTM？

**你的回答摘要**：self-attention 可以同时读取所有文字；有多层（浅层/中层/深层）；LSTM 是线性的很慢；LSTM 上下文太长会"搞忘"。

**评分：8/10** — 核心都答到了，两个层面的优势都提到了。

**扣分点**：

- "多层"不是 Transformer 独有的优势，LSTM 也可以叠多层。所以这不是取代的原因
- "同时把所有文字转换成 token"表述不准确——tokenization 是 Transformer **之前**的步骤，不是 self-attention 做的事。Self-attention 做的是让已经变成 token 的序列中，所有 token **互相看对方**，计算关联度

**参考答案**：

1. **工程层面**：Transformer 可以并行计算，LSTM 必须顺序处理 → 训练更快、能做更大的模型
2. **架构层面**：Self-attention 让任意两个 token 直接建立关联，不受距离限制。LSTM 信息逐步传递，距离越远越容易丢失（你说的"搞忘"）

### Q2. "Attention Is All You Need" 标题的含义

**你的回答摘要**：attention 指 self-attention，可以一次性把所有文字转换成 token。

**评分：4/10** — 知道 attention 指 self-attention，但标题含义没答出来。

**核心问题**：self-attention 不是"把文字转成 token"，那是 tokenizer 干的。

**参考答案**：作者在说——我们**不需要** LSTM 那些复杂的门控递推机制，**只用**一个更简单的 self-attention 机制**就够了**。标题的潜台词是"简化反而赢了"，因为简化换来了并行能力和规模优势。"All You Need" 强调"这就足够了，不用搞那么复杂"。

**你的回答摘要**：prompt engineering 是规定 LLM 的前提条件；context engineering 包含了 prompt engineering，还有对话历史、RAG、用户记忆偏好等。

### Q5. Training time scaling vs inference time scaling

**你的回答摘要**：inference time scaling 是模型不变，通过调参数让效果更好；training time scaling 是模型越大参数越多预测越好；正交 = 两个一起用。

**评分：6/10** — 大方向对，但有一个关键误解。

**关键错误**：Inference time scaling **不是"调参数"**。模型的参数在推理时是**固定不变的**。它是通过**使用方式**的技巧来提升表现：

- Reasoning：让模型先生成思考过程再回答
- RAG：往输入里塞更多参考信息
- Multi-shot prompting：在输入里给多个示例

这些都不改变模型参数，只改变了输入内容或生成方式。

### Q9. "exquisitely" vs "the" 的 token 数

**你的回答摘要**：exquisitely 大概 4-5 个，the 是 1 个；常见词 1 个 token，罕见词 3-5 个。

**评分：8/10** — 数字和规律都对。

**扣分点**：你说"平均一个 token 是四个单词"然后自己也说搞不懂——弄反了。是 **1 token ≈ 4 个字符（字母）**，不是 4 个单词。另一个换算是 1000 token ≈ 750 个单词。

---

### Q10. 三位数准、四位数不行

**你的回答摘要**：每三个数字一个 token；三位数在训练数据里有；四位数的 token 不知道是什么。

**评分：8/10** — 核心逻辑对。

**更精确的说法**：不是"四位数的 token 不存在"，而是四位数被拆成了**两个 token**（如 1234 → "12" + "34"）。三位数加法是 1 个 token + 1 个 token = 1 个 token，模型可以直接记忆；四位数加法跨了多个 token，模型需要真正理解进位逻辑，早期模型做不到。

## 需要巩固的 3 个点

1. **Self-attention ≠ tokenization**：你多次把 self-attention 描述成"把文字转成 token"。Tokenizer 负责文字→token，self-attention 负责让 token 之间互相计算关联度。这是两个不同的步骤
2. **Inference time scaling ≠ 调参数**：参数在推理时是固定的。Inference time scaling 是通过 reasoning、RAG 等**使用方式**的技巧来提升表现
3. **数量级别搞混**：1M = million = 100 万，1B = billion = 10 亿，注意别混
