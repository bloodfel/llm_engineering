## 概念理解题

**Q1.** Transformer 为什么取代了 LSTM？只说"更快"不算完整答案，至少说出两个层面的优势。

我回想一下啊。

第一个是 Transformer 有个 self-attention 的机制，它可以同时读取所有的文字，并且它不只是一层，啊，它是有多层的，比如说有浅层含义、中层含义，还有深层含义。浅层呢，只专注于语法，中层呢，就专注于其他的。可以一次性把所有的文字全部，呃，转换成那个 token 的形式，呃，再来预测下一个，呃，词。

呃，LSTM 呢，必须要先把第一个词接收了过后，再来预测下一个，然后把第二个词接收了过后，再来预测第三个，叫——它是线性的这个预测，很慢。嗯。

然后呢，第二个部分呢，第二个层面我觉得应该说得差不多了吧。就是第一个是平行的，第二个是可以多层的。并且，呃，当 LSTM 它上下文太长的时候，它可能会搞忘，呃，最开始的那些记忆。但 Transformer 因为它是——同时把所有的文字全部输入进去，就不会存在搞不搞忘了这个这一说。

**Q2.** "Attention Is All You Need" 这个标题想表达什么意思？attention 在这里指的是什么机制？

Attention 指的是 self-attention 吧，就是注意力机制，就是可以一次性地把所有的文字全部转换成 token 的形式。

**Q3.** 什么是 emergent intelligence？用你自己的话解释为什么"预测下一个 token"居然能产生智能。

就是 emergent intelligence 就本来，呃，这是个概率模型，统计学。

但是当这个训练量达到一定数量的时候，就会产生智能。就比如说有一些没有预训练过的东西，它也能够预测。呃，只要当——这个训练量达到足够大的时候，就举个例子，呃，像一只蚂蚁，它不明白为什么，不明白怎么能够协同工作。

但是一群蚂蚁，一非常大的一群蚂蚁就能够搞明白相互之间如何协同工作，来组成这种蚂蚁社群，还有工种的分配。它一个蚂蚁是绝对没有这种智力的。

但当量大到一定数量的时候，就会出现这种 emergent intelligence。

**Q4.** Context engineering 和 prompt engineering 有什么区别？举一个具体场景说明 context engineering 在做什么。

Prompt engineering 是规定了人工智能 LLM 的先情条件，就是规定它你是一个什么样的人，你应该做什么样的事，你的回复应该是什么。

Context engineering 包含了 prompt engineering，然后还有一些其他的东西，就是上下文工程。比如说，包含了用户之前的对话，包含了，呃，信息库，就是 RAG。嗯。然后包含了用户的——对于用户的记忆和用户的偏好等等等等。

就是能够让——然后让他们在一次回答中充分理解上下文之后，再能够进行下一次的这个预测。

**Q5.** Training time scaling 和 inference time scaling 分别是什么？为什么说它们是"正交的"？

我记得 **Inference Time Scale** 是在模型大小不变的情况下，通过调参数等方式，让模型能够获得更好的预测效果。

**Training Time Scale** 就是模型越大，参数越多，输入越多，它的预测能力就更好。

正交的就是可以两个一起用。在更大的参数和更大的输入的情况下，我们对模型的参数进行微调，从而让它在一个有限的模型参数框架内，能够给出更好的下一个 token 的预测。

## Token 与计费题

**Q8.** 为什么 LLM 不直接用"一个单词 = 一个输入"的方案，而是用 token（子词片段）？Token 方案的核心优势是什么？

哦，因为这样的话，我记得是会消耗，就是这个单词库会特别特别量巨大。

Token 方案的核心优势是，假设遇到了像比如 ChatGPT 这种新的词汇，它可以分为两个或者三个 token，然后再来理解，并且可以减少，呃，单词输入量。

比如说一个 "playing" 和 "walking"。它可以分为 "play" 加 "ing" 两个 Token。那么再加上 "work"，W O R K，再然后再加上之前有的 "ing"，这两个又能够组成一个新词。

那么从原来的四个词到现在我们只需要三个词，三个 Token，就节省了这个，啊，单词的这个储存量。

**Q10.** 早期 GPT 算三位数加法很准，但四位数就不行。这和 token 有什么关系？

因为早期 ChatGPT，每三个数字都把它转换成一个 token。

那么，它是用这每三个数字转换成一个 token 这种框架来训练 ChatGPT 的，所以说它的三位数加减法很准。

但四位数，它不知道一个四位数的 token 是什么，而且没有被训练过。

所以，它的四位数加减法或乘除法就很差。

**Q11.** 从你输入一条消息到收到 ChatGPT 的回复，中间经历了哪些步骤？按顺序列出来（至少 5 步）。

ChatGPT 会第一步，收到你的这个消息过后，会把这些消息 tokenize 成 token。

第二步，会把这些 token……啊，我想一下。喂给 transformer。

Transformer 呢，会把所有 token 接收到，然后预测下一个 token，然后把下一步把新的 token 拼回去，然后再预测下一个。重复上述过程，直到有个叫结束的标记。
