## 核心论点

### 1. KV Cache：避免重复计算

LLM 是**自回归（autoregressive）**的，逐个生成 token。每生成一个新 token 都要看前面所有 token 的 K 和 V，但前面 token 的 K、V 每次算出来都一样，重复计算是浪费。

**KV Cache 的做法：** 每个 token 的 K 和 V 只算一次，存起来，后面直接复用。

```
第 1 步（处理 prompt "我今天吃了一碗"）：
  → 算出所有 token 的 K、V → 存入 KV Cache

第 2 步（生成"热"）：
  → 从缓存取 prompt 的 K、V（不重新算）
  → "热"过模型 → 得到 Q_热、K_热、V_热
  → Q_热 和缓存里的 K 算 Attention → 输出概率分布 → 选出"热"
  → K_热、V_热 存入缓存

第 3 步（生成"干"）：
  → 从缓存取所有历史 K、V
  → 只算"干"的 Q、K、V → Attention → 选出"干"
  → K_干、V_干 存入缓存
```

**KV Cache 和 Prompt Caching 的关系：**
- **KV Cache** — 单次请求内部，避免重复计算已生成 token 的 K、V
- **Prompt Caching** — 跨请求，把 KV Cache 持久化到服务器，下次同样前缀的请求直接复用

---

### 2. 生成顺序：为什么必须一个一个生成

模型的设计是自回归的：每一步的输入依赖上一步的输出。

```
P("热" | "我今天吃了一碗")            → 先算这个概率，选出"热"
P("干" | "我今天吃了一碗热")          → 再算这个，选出"干"
P("面" | "我今天吃了一碗热干")        → 再算这个，选出"面"
```

如果 token 1 是"白"而不是"热"，token 2 可能就是"米"而不是"干"。不知道上一步的结果，就没法算下一步。

类比：写作文一个字一个字写，写了"热"才知道下一个可能是"干"。

---

### 3. Attention 核心公式

```
Attention(Q, K, V) = softmax(QK^T / √d) × V
```

四个步骤：

**第 1 步 `QK^T`：算相似度（点积）**

当前 token 的 Q 和所有前文 token 的 K 做点积。方向越一致，点积越大，因为点积本质上在算两个向量夹角的余弦：`a · b = |a| × |b| × cos(θ)`。

**第 2 步 `/ √d`：缩放**

除以向量维度的平方根。不缩放的话，维度越高点积数值越大，softmax 会变得极端（几乎 100% 关注一个 token）。缩放让注意力分布更平滑。

**第 3 步 `softmax(...)`：变成权重**

把分数变成概率（加起来 = 1），表示对每个前文 token 的关注程度。

**第 4 步 `× V`：加权求和**

用权重乘以对应的 V，加起来。结果是一个融合了前文信息的向量，重点偏向权重高的 token。

---

### 4. Q、K、V 是什么

一个 token 进入模型后，乘以三个不同的权重矩阵，**同时**产出 Q、K、V 三个向量：

- **Q (Query)** — 当前 token 的"提问"：我在找什么？当场用完就扔
- **K (Key)** — 这个 token 的"标签"：我是什么？留给后面的 token 用（存入 KV Cache）
- **V (Value)** — 这个 token 的"内容"：如果被关注，我提供什么信息？留给后面的 token 用（存入 KV Cache）

Q 是当场用的（和前面的 K 算相似度），K 和 V 是留给未来的 token 用的。

---

### 5. Attention 的输出 ≠ K 或 V

容易混淆的三个东西：

| 向量 | 是什么 | 什么时候产生 |
|------|--------|------------|
| **Embedding** | token 的初始表示，查表得到，不管上下文 | 最开始 |
| **Q、K、V** | 从 embedding 变换出来的中间工具 | Attention 计算时 |
| **Attention 输出** | 所有 V 加权求和的结果，融合了上下文 | Attention 计算后 |

Attention 输出不是某一个 token 的 V，而是**所有 V 的加权混合物**。K 和 V 是工具，Attention 输出是用工具算出来的成品。

同一个词"吃了"，embedding 永远一样，但 Attention 输出随上下文而变：
- "我今天吃了一碗" → Attention 后 → "吃了"的向量偏向"进食"
- "他吃了一惊" → Attention 后 → "吃了"的向量偏向"受到/经历"

**Attention 的意义：把一个"死"的词向量，变成一个"活"的、带上下文的向量。**

---

### 6. Attention + MLP = 一个 Transformer Block

每个 Block 包含两个部分，分工明确：

**Attention = 收集信息（看题目）**
- 让每个 token "看到"其他所有 token
- 输出：每个词都带上了上下文信息
- 类比：考试时读题目，把所有条件收集起来

**MLP = 加工推理（用知识解题）**
- 两层矩阵乘法 + 激活函数：先扩大维度（如 4096 → 16384），再缩回来
- W₁、W₂ 矩阵里存的是训练时学到的知识和模式
- 类比：用学过的知识去处理收集到的信息

```
Attention 告诉我：主语是"我"，动作是"吃了"，量词是"一碗"
  ↓
MLP 用学到的知识处理：
  "吃了一碗" → 激活"食物"相关的模式
  → 向量被推向"食物类词汇"的方向
```

---

### 7. 多层 Block 的叠加

GPT-4 级别的模型有 100+ 个 Block 串联。结构相同，但每层的参数（权重矩阵）不同，是训练出来的自然分工：

```
Block 1（浅层）：学语法、词性、简单搭配
Block 2-3（中层）：学语义、短语关系
Block 50+（深层）：学复杂推理、长距离依赖
```

类比：读一篇文章读很多遍，每一遍做的事"看起来一样"（都是读），但每遍关注点不同——第一遍看字面意思，第二遍看逻辑关系，第三遍看深层意图。层数越多，理解越深。

---

### 8. 从输出向量到选词

最后一个 Block 的输出向量不是直接选词，还要经过：

```
最后一层输出 [4096 维]
  ↓ linear 层（乘以 embedding matrix 的转置）
  ↓
  [10 万维] — 词表里每个 token 一个分数
  ↓ softmax
  概率分布：["的"=1%, "干"=85%, "面"=3%, ...]
  ↓
  选分最高的 → "干"
```

**为什么乘以 embedding 转置就能得到分数？** 因为矩阵乘法 = 批量点积。输出向量和词表里每个 token 的 embedding 做点积（算相似度），一次矩阵乘法就把 10 万次点积全算完了。转置只是为了让矩阵维度对齐。

有趣的是：入口和出口用的是**同一张 embedding 表**。进来时查表把 token 变成向量，出去时用同一张表反过来把向量变回 token。

---

### 9. Embedding 表

Embedding 表是一个 `100000 × 4096` 的矩阵。10 万行（每个 token 一行），每行 4096 个数字。

这 4096 个数字是训练出来的，人类无法直接解读每一维的含义。但训练的结果是意思相近的词，向量方向相近：

```
"面"的向量 和 "饭"的向量 → 方向接近（都是食物）
"面"的向量 和 "椅子"的向量 → 方向差很远
```

---

### 10. 点积为什么能衡量相似度

点积公式等价于：`a · b = |a| × |b| × cos(θ)`

- 方向一致 → θ = 0° → cos = 1 → 点积最大
- 垂直 → θ = 90° → cos = 0 → 点积为零
- 方向相反 → θ = 180° → cos = -1 → 点积为负

Transformer 里所有的"算相似度"（Q 和 K 的点积、输出向量和 embedding 的点积）都是利用这个数学性质。

## 关键术语

| 术语 | 解释 |
|------|------|
| **自回归 (Autoregressive)** | 逐个生成 token，每步依赖上一步的输出 |
| **KV Cache** | 缓存已计算的 K 和 V，避免重复计算 |
| **Q (Query)** | 当前 token 的"提问"向量，用来找相关的前文 |
| **K (Key)** | token 的"标签"向量，让别的 token 能找到自己 |
| **V (Value)** | token 的"内容"向量，被关注时提供的信息 |
| **点积 (Dot Product)** | 对应位置相乘再相加，衡量两个向量的方向一致性 |
| **softmax** | 把一组分数变成概率分布（加起来 = 1） |
| **√d 缩放** | 除以维度的平方根，防止 softmax 过于极端 |
| **Transformer Block** | Attention + MLP 组成的基本单元，模型由多个 Block 串联 |
| **MLP (Multi-Layer Perceptron)** | 多层全连接网络，存储训练学到的知识，做加工推理 |
| **Embedding** | token 到向量的映射表，固定大小（如 100000 × 4096） |
| **linear 层** | 最后一步，把输出向量映射成词表大小的分数 |
| **激活函数** | 引入非线性，让网络能学习复杂模式（如 ReLU：负数变 0） |
